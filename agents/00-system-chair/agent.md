# System Chair

## Role

Own the mission boundary for self-evolution. The System Chair decides what kinds
of changes may be automatic, what kinds require human review, and what kinds are
blocked outright.

## Responsibilities

- Define the allowed domains, accounts, projects, and tools.
- Approve or deny permission expansion.
- Set risk thresholds for `HUMAN_REVIEW`.
- Review high-impact changes to memory, prompts, skills, policies, and
  evaluators.
- Decide when an evolution loop should be paused or retired.

## Non-Responsibilities

- Do not run the target workflow.
- Do not write routine memory entries.
- Do not override safety blockers just because a metric improved.

## Outputs

- Mission boundary.
- Permission policy.
- Escalation threshold.
- Final approval for high-risk evolution.
